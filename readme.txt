Тестовое задание - HTML-верстальщик
Требуется сверстать только первый блок страницы (шапка). Ниже подробнее о каждом блоке:

Шапка (rs-menu-cart)
Ховер для пунктов меню показан в макете
Требования
Разработку требуется делать на основе блоков из нашей HTML-вёрстки. Использования своих блоков запрещается.
Стек
html 5, css 3, js
bootstrap 3
Font Awesome 4
jquery 3.2

другие технологии использовать запрещается

CSS и Блоки
Все блоки должны быть обернуты в контейнер. Например:

<div class="navbar-001">
  <div class="container">
   <div class="row">
    <div class="col-md-12">
     <p>...</p>
     <div>...</div>
    </div>
   </div>
  </div>
 </div>

Всем классам CSS указываем родителя из таблицы блока по вложенности. Например:

 .navbar-001 {
  font-size:16px;
  font-weight:100;
 }
 .navbar-001 .menu {
   padding:0px;
 }
 .navbar-001 .navbar-inverse .navbar-toggle {
  border-color: #fff;
 }
 .navbar-001 .navbar-inverse .navbar-toggle .icon-bar {
  background-color: #fff;
 }
 .navbar-001 .navbar-inverse .navbar-toggle:hover,
 .navbar-001 .navbar-inverse .navbar-toggle:focus {
  background-color: #254a7c;
 }

Использование Less/Sass не требуется
Должна быть предоставлена полная, не минифицированная версия css файла
CSS должен содержать только код, относящийся к блоку. Все неиспользуемые css стили должны быть удалены.
Стили не должны быть привязаны к id. Использование !important запрещается.
Блочная вёрстка: каждый блок должен иметь свой индивидуальный css файл, блоки должны быть визуально отделены друг друга в коде. В каждом css файле должны быть стили, относящиеся только к одному блоку.
В файле со скриптами должны присутствовать краткий комментарий, описывающий к какому блоку принадлежит та или иная часть кода, и что он делает. Например: “//слайдер для rs-slider”, “//rs-text-block на главной, калькулятор”, “//rs-text-block на странице контакты, api яндекс карты”

Адаптив
Адаптив настраивается при помощи bootstrap 3.4 (col-xs) и @media запросов css (max-width). 
Сайт должен корректно отображаться:
на любой ширине экрана (блоки должны отображаться корректно при плавном уменьшении ширины окна с 3840px до 360px).
на любых разрешениях, включая UltraWide мониторы.
в мобильной версии, включая landscape
на всех популярных браузерах: Chrome, Opera, Firefox, Safari, IE 11
Проверка
Вёрстка должна соответствовать макету. Допускается погрешность в отступах из-за адаптивности экрана до 5%
Валидатор HTML (верстка должна быть без ошибок);
Семантика верстки (один h1 на странице. Заголовки h2-h4, текст в <p> и т.д.)
